# HVAC 沉浸式 3D 展示版设计规格

## 目标

在完整保留初始版本的前提下，将中央空调水冷系统升级为以实际设备结构为核心的可交互 3D 展示。首屏必须直接呈现可旋转的 WebGL 场景；用户可观察水流、设备运转与制冷循环，也能把场景作为未来软件中的独立视图嵌入。

## 原版保留

初始提交中的 `index.html`、`styles.css`、`src/main.js`、`package.json`、`package-lock.json`、`README.md` 和原效果图原样复制到 `original/`。新版继续使用仓库根入口，原版可通过在 `original/` 目录单独启动 Vite 查看。

## 视觉方向

场景采用工业展厅而非传统浅色教学沙盘。背景为中性深灰，设备主体使用米白控制柜、深海军蓝换热器、镀锌钢、黄铜与橙色安全件，四类水管保持蓝、靛、青绿、橙四种工程识别色。照明模拟机房线性灯和设备展陈聚光，地面只保留服务于空间定位的细网格与设备基础。

本轮展示首屏改为 MAU 电影化剖视数字孪生，不再默认展示全厂沙盘。MAU-01 采用接近正面的轻透视镜头，占桌面端有效画面约 70%，地面、网格、主干管网及其他设备全部隐藏；原有完整机房保留在“系统全景”模式中。深午夜蓝背景、冷白金属外壳与高饱和介质光效形成层次，配色使用 `#07111d`、`#d7e0df`、`#18c8ff`、`#ff4d52`、`#45df8a` 和 `#e857ff`。

MAU 外壳在首屏默认半透明，检修框架和内部功能段保持实体。设备内部形成一条连续动画气流：入口与加热区域为热红色，穿过冷却盘管后过渡为冰蓝色，加湿与送风区域为青色。气流由密集发光粒子、局部辉光舱段和流向速度共同表达，不使用参考图静态贴片。冷却盘管下方外露 CHWS/CHWR 支路、控制阀、平衡阀和执行器，阀门状态与水流粒子同步显示。

首屏 HUD 只显示理解 MAU 工况所需的数据：新风温湿度、过滤器压差、冷冻水阀位、送风温度、送风湿度、风量和风机状态。桌面端数据分布在设备两侧，移动端压缩为上下信息条，不遮挡主要内部组件。参考图仅用于镜头、剖视、材质和介质流动语言，不复制其商标、水印或具体控制界面。

首要视觉对象是螺杆式水冷冷水机组。外观参考用户提供的实物图，包含双层壳管式蒸发器/冷凝器、法兰端盖和螺栓、上置控制柜、半封闭螺杆压缩机、U 形吸排气管、黄铜旁通管、压力表、铭牌、支座和底座。设备比例不绑定具体厂商型号，不复制图片商标或水印。

## 运行原理

默认“系统运行”模式同时展示四条水路：冷冻供水 CHWS、冷冻回水 CHWR、冷却供水 CWS、冷却回水 CWR。流动粒子沿正确方向运动，泵联轴器和冷却塔风机持续旋转，温度颜色在蒸发器与冷凝器附近形成可见过渡。

“制冷循环”模式聚焦冷水机组，将压缩、冷凝、节流、蒸发四阶段按真实顺序高亮。压缩机切面内的双螺杆转子转动，非当前部件降低饱和度。模式切换只改变显示层级和镜头焦点，不重建整个场景。

“设备透视”模式使用类似汽车透明车身的表达：选中设备的外壳降低至约 12% 不透明度，内部工作件保持实体材质并加入边缘高亮。冷水机组显示壳管换热器的管板与铜管束、双螺杆转子、制冷剂循环和水侧流向；水泵显示蜗壳内叶轮和吸排水路径；冷却塔显示布水管、填料、下落水滴、进风与上升气流；MAU 显示新风入口、初效过滤器、冷却盘管、加热段、加湿段、送风机和出风段。透视模式可与设备点选结合，不影响全厂管网状态。

通用设备透视采用统一三层结构：设备外壳降至 8%–12% 不透明度，机械工作件和剖切边缘保持实体，水、风和制冷剂以半透明体积层与定向粒子表达。透视模式隔离当前设备、隐藏地面和其他机组，同时保留透明管网。冷水主机的蒸发器与冷凝器必须显示半剖壳体、管板、挡板环、管束和壳程介质体，压缩机显示剖切机壳、转子腔、定子与轴承；水泵显示蜗壳水体、离心叶轮、轴、机械密封、轴承和电机定子；冷却塔显示热水布水、喷淋锥、PVC 填料、收水器、落水、盆水、上升气流和轴流风机。

全厂管道采用“透明管壁 + 内部流芯 + 定向胶囊粒子”表达。全景模式管壁保持约 48% 不透明度，透视模式降至约 14%；内部水体至少占管径 60%，透视时约 65% 不透明度。粒子颜色由对应 CHWS、CHWR、CWS、CWR 工程色提亮得到，并沿管线切线方向运动。

MAU 采用数字孪生剖视表达：概览模式显示完整金属机柜和检修门，设备透视模式将侧板和顶板淡出，内部按“防雨百叶、G4 初效、F8 中效、表冷除湿、再热、蒸汽加湿、送风机、消声送风”顺序保持可见。空气粒子单次穿越控制在约 11–13 秒，各段使用独立框线与色带，颜色只在跨越功能段时变化。蓝色冷冻水粒子通过盘管，送风侧颜色随温湿度状态轻微变化。面板显示新风温湿度、盘管阀位、过滤器压差、送风温度和风量。

全厂模式参考工业监控画面的信息能力，但不复制大面积悬浮框。压力、温度、流量、阀位和功率数据只在所选系统或设备附近出现，使用固定尺寸的 DOM 标签避免随相机透视变形。管内以沿曲线运行的亮带与粒子共同表达流向，粒子速度与显示流量成比例。

## 交互

- 鼠标拖动或单指旋转，滚轮或双指缩放。
- 默认进入“MAU 剖视”模式并自动选中 MAU-01；自动巡航、全厂管网和普通设备标签默认关闭。
- “MAU 剖视”与“系统全景”使用独立镜头预设，切换时平滑过渡，不重建场景。
- 点击设备打开右侧检查面板，显示设备名称、状态、供回水温度、流量或效率等示例运行值。
- 顶部仅显示项目名称、运行状态和负载；底部用图标按钮提供自动巡航、运行原理、设备透视、管网显隐、数据标签显隐和重置视角。
- 点击空白区域关闭设备面板；按 Escape 同样关闭。
- 在窄屏上，检查面板变为底部抽屉，控制栏保持一行可横向滚动，3D 场景始终占满视口。

## 嵌入边界

设备数据、运行模式、内部组件层和场景创建拆分为独立 ES 模块。入口只负责装配，宿主软件未来可通过 `window.HVACShowcase` 调用 `setMode()`、`focusEquipment()`、`setXray()`、`resetView()` 和 `getState()`。所有运行数据先使用本地确定性数据，不依赖服务端或密钥；设备构建器保留加载 GLB 的替换边界。

## 性能与可访问性

桌面端像素比上限为 1.75，移动端为 1.35；重复螺栓和粒子共享几何体与材质。页面尊重 `prefers-reduced-motion`，关闭自动巡航并降低粒子速度。所有按钮有可读标签和焦点状态，信息不只依赖颜色表达。

## 验证标准

- `npm test` 覆盖模式状态、设备清单、水路方向和公开控制 API。
- `npm run build` 成功，无未解析模块。
- Playwright 在 1440x900 与 390x844 下确认页面无横向溢出、控制不遮挡设备面板、WebGL 画布非空。
- 默认首屏只能看到 MAU-01；MAU 外壳透明、所有功能段可辨识，冷热气流和盘管水流持续运动，CHWS/CHWR 阀组可见。
- 自动巡航、模式切换、设备透视、管网显隐、设备选择和重置视角均在真实浏览器中执行。
- 桌面与移动端各保存一张实际渲染截图，并写入作者元数据 `yewwung`。
