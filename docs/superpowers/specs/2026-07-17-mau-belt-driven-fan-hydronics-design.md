# MAU 蜗壳离心风机与水系统状态设计规格

## 目标与范围

保留 MAU-01 的 100% 新风工艺和八段处理顺序，将现有直驱 EC 插入式离心风机重建为参考图中暖通常见的单进风、皮带传动蜗壳离心风机。风机必须在剖视状态下一眼可辨识蜗壳、圆形进风口、后弯叶轮、外置电机和皮带传动关系，并通过局部气流动画表达离心风机的轴向吸入、叶轮增压和切向排出过程。

同步强化表冷盘管水系统：在设备侧明确显示 CHWS/CHWR 管道、阀门、流向、流动状态和阀门开度。状态信息既要在 3D 设备附近可读，也要保留网页 HUD 汇总，并形成可由宿主软件替换的状态数据边界。

本轮不增加回风机、混风段和排风段，不将 MAU 改为完整 AHU，不修改冷水机组、水泵、冷却塔、系统总管和 `original/` 快照。

## 方案选择

### 方案 A：Three.js 程序化高细节模型（选定）

在现有 `createSupplyFan()` 边界内重建风机。优点是无需外部 3D 资产即可直接运行和嵌入，部件命名、动画、拾取和实时状态均可测试；同时保持模型为独立组件，后续可用同名 Blender GLB 替换。缺点是曲面精度和烘焙材质不及专业 DCC 资产，需要通过合理轮廓、分层材质和灯光控制提高真实感。

### 方案 B：Blender 建模后导出 GLB

可获得更精细的钣金、圆角和烘焙材质，但当前环境没有 Blender，也没有可复用的风机资产；立即引入会增加建模、导出、加载和版本管理链路。本轮不采用，保留为后续生产级资产升级路线。

### 方案 C：完整双风机 AHU

按参考图增加回风机、混风段和排风段，结构最接近参考图，但会改变当前 MAU 的系统定义、机组尺寸和气流逻辑。本轮不采用。

## 风机结构

### 蜗壳与进风侧

- 风机根节点继续使用 `supply-fan`，整体安装在现有送风段内。
- 使用挤出轮廓构建具有真实扩压趋势的蜗壳侧板和环形壳体，蜗舌位于切向出风口附近；壳体采用镀锌钢板，前侧局部透明或开盖以显示内部叶轮。
- 进风侧设置圆形喇叭口、加厚检修环、均布螺栓和深色叶轮眼，避免再次出现插入式风机的方形支架外观。
- 蜗壳顶部或侧上方形成矩形切向出风口，并通过短扩压段接入 MAU 下游静压箱，气流不能水平穿过叶轮侧面。

### 叶轮与传动

- `supply-fan-rotor` 包含前后轮盘、中心轮毂和不少于 10 片有轴向深度的后弯叶片，叶轮使用参考图类似的工业红色，壳体使用中性金属灰以形成结构对比。
- 风机轴从叶轮延伸至传动侧，具有轴承座、轴承端盖和连接法兰。
- 外置 TEFC 电机安装在可调滑轨上，显示电机壳体、散热筋、端盖和接线盒。
- 电机小皮带轮与风机大皮带轮之间设置两条 V 带，并覆盖半透明金属网防护罩；防护罩在剖视模式下仍可看清传动关系。
- 风机、电机和传动安装在共同槽钢底座上，底部设置四个弹簧或橡胶减振器。

### 动画关系

- 叶轮、风机大皮带轮、电机小皮带轮保持正确传动比；V 带不做逐点物理求解，但通过纹理或高亮标记的循环位移表达运行。
- 风机频率使用当前演示值 `42.6 Hz`，叶轮转速与频率成比例。停止状态时叶轮、皮带和局部风机气流同时停止。
- 主 MAU 气流保持当前慢速节奏。风机局部轨迹分为轴向收敛进入叶轮眼、随叶轮旋转增压、切向进入蜗壳和经出风口进入下游静压箱四段。

## 冷冻水阀门与状态

### 设备构成

- CHWS 支路保留电动两通调节阀，显示阀体、法兰、阀杆、执行器和本体位置指示器，默认开度 `63%`。
- CHWR 支路保留静态平衡阀，增加刻度手轮和锁定环，显示默认设定开度。
- 供回水侧各保留压力表，并增加温度测点；供水以蓝色、回水以青色区分，颜色仅用于状态编码，不做大面积装饰发光。
- 盘管内部和外部管路继续使用透明管壁与定向流动粒子，箭头和粒子方向必须符合供水进入盘管、回水离开盘管的实际路径。

### 运行状态表达

- 3D 设备附近增加紧凑状态牌，至少显示：`CHWS 7.0 C`、`CHWR 12.0 C`、瞬时流量、阀门开度和 `流动/停止/故障` 状态。
- 阀门执行器增加开度环或位置指针。`0%` 时关闭且水流粒子停止；`1%-100%` 时粒子速度和亮度随开度分段变化，避免低开度时看起来仍为满流量。
- 正常流动使用蓝/青色，停止使用低亮度灰蓝，故障使用局部红色状态标识；故障色不覆盖整段管道。
- HUD 中的阀门开度和水系统状态使用同一份状态数据，避免 3D 标签与界面数值不一致。

## 数据与嵌入边界

MAU 状态集中为一个确定性对象，默认值供离线演示使用：

```js
{
  fanRunning: true,
  fanFrequencyHz: 42.6,
  chwsTemperatureC: 7.0,
  chwrTemperatureC: 12.0,
  chilledWaterFlowM3h: 8.6,
  controlValveOpeningPct: 63,
  balancingValveOpeningPct: 78,
  waterFlowState: "flowing"
}
```

状态对象驱动叶轮/皮带转速、水流粒子、阀位指针、3D 状态牌和 HUD。现有 `window.HVACShowcase` API 保持兼容，并新增小范围 MAU 状态更新入口；输入值进行范围限制，未知字段被忽略，不影响现有模式切换和设备聚焦。

## 视觉与镜头

- 默认剖视镜头继续完整展示八段 MAU，不让风机局部放大破坏整机比例。
- 风机段使用局部检修开盖/半透明壳体，而不是让整台风机消失；叶轮红、壳体灰、电机深蓝、皮带黑，确保部件层次明确。
- 水阀状态牌放在盘管水管附近，并使用引线指向对应部件；标签尺寸稳定，在桌面和移动端不与设备段标题重叠。
- 选中风机或阀门时，现有设备聚焦与标签系统继续工作，不新增营销式说明卡片。

## 兼容与文件边界

- 保持 `buildMau(options)`、`group.userData.animation` 和现有设备 ID 兼容。
- 保留 `supply-fan-rotor` 名称供统一动画循环驱动，新增皮带轮、传动和阀位节点使用稳定、可测试的名称。
- 风机几何仍在 MAU 设备模块或其专用子模块内，不影响其他设备构建器。
- 不修改 `original/`，不删除现有公开 API，不引入需要网络加载的运行时资产。

## 测试与验收

自动测试先更新为失败状态，再完成实现，至少验证：

- 风机包含蜗壳侧板、圆形进风环、检修螺栓、切向出风口、红色后弯叶轮、风机轴、轴承座、外置电机、大小皮带轮、双 V 带、防护罩、共同底座和四个减振器。
- 叶轮包含不少于 10 片有非零轴向深度的后弯叶片。
- 叶轮轴线与 MAU 主气流方向一致，切向出风口通向下游静压箱。
- MAU 风机气流轨迹包含轴向吸入、旋转增压、切向排出和下游恢复四阶段。
- CHWS 调节阀、CHWR 平衡阀、阀位指针、温度/压力测点和供回水流动路径存在。
- 状态更新能够同步改变风机转速、水流动画、阀位和显示值，且 `0%` 阀位会停止水流动画。
- 原有 MAU 分段、系统总览、设备选择、嵌入 API 和 `original/` 保持不变。

Playwright 在 1440x900 和 390x844 下进行实际浏览器验收：

- 风机在整机剖视中可立即辨识为带蜗壳的离心风机，不再像轴流风机或插入式风机。
- 圆形进风口、红色叶轮、外置电机、皮带和防护罩能同时辨识，动画运行时无穿模或不合理旋转。
- 水流方向、阀门开度和供回水状态清楚可读，3D 标签与 HUD 数值一致。
- 主气流保持慢速，MAU 各段边界清晰，桌面和移动端均无文字重叠或画布空白。
